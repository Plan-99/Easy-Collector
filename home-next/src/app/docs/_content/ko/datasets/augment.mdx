증강(Augmentation)은 같은 에피소드를 여러 변형으로 늘려 학습의 일반화 성능을 높이는 기법입니다.

<Callout type="info">
**증강은 [일괄 변경 (Batch Edit)](/docs/datasets/batch-edit) 패널의 한 섹션으로 통합되었습니다.** 기존 풀스크린 *Data Augmentation Dialog* 는 제거되었고, 데이터셋 우클릭 메뉴의 *Augmentation* 항목도 **Batch Edit** 항목으로 합쳐졌습니다.
</Callout>

## 새 위치

좌측 네비의 **데이터셋 탭**에서 다음 중 하나로 BatchEditPanel을 열고 **Augmentation** 섹션을 펼치세요.

- 트리에서 **데이터셋 폴더 라벨**을 클릭
- 데이터셋 우클릭 → **Batch Edit**

## 무엇이 같고 무엇이 달라졌나

| 항목 | 예전 (Data Augmentation Dialog) | 현재 (BatchEditPanel → Augmentation) |
|---|---|---|
| Lightness / HSV / Disturbances / Salt & Pepper / Gaussian / Perspective | ✅ | ✅ (동일) |
| 반복 횟수 (Repeat) | 다이얼로그 안 입력 | 섹션 상단의 **Repeat** 필드 (1~20) |
| Crop / Rotate 같이 적용 | ✘ | ✅ (같은 패널의 별도 섹션) |
| 출력 데이터셋 이름 | 다이얼로그 끝의 입력 | 패널 상단 출력 이름 입력 (기본값 `<원본>_transformed`) |
| 진행률 이벤트 | `augmentation_progress` | `transform_progress` (구 이벤트도 호환 유지) |
| 백엔드 엔드포인트 | `POST /dataset/<id>/augment` | `POST /dataset/<id>/transform` (구 엔드포인트는 wrapper로 유지) |

증강 옵션 자체의 의미와 슬라이더 범위는 그대로입니다. 자세한 설명은 [일괄 변경 → Augmentation 섹션](/docs/datasets/batch-edit#augmentation-섹션)을 참고하세요.

<Callout type="tip">
증강은 새로운 에피소드 파일을 생성합니다. 원본은 그대로 유지되니 안심하고 실행하세요. `repeat=N`이면 원본 1개당 N개의 무작위 변형 에피소드가 만들어집니다.
</Callout>
