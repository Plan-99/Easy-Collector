다운샘플은 원본을 건드리지 않고 **새 데이터셋을 만들어** 에피소드 프레임을 솎아내는 작업입니다. 30Hz로 수집한 데이터를 10Hz 정책 학습에 쓰고 싶거나, 시퀀스가 너무 길어 학습 시간이 부담스러울 때 사용합니다.

<Callout type="info">
원본 데이터셋은 그대로 남고 결과는 `<원본이름>_downsampled` 라는 새 데이터셋에 저장됩니다. 재생 속도는 그대로(fps 유지), 프레임 수만 줄어듭니다.
</Callout>

## 어디서 하나

기본 진입점은 좌측 네비의 **데이터셋 탭**이고, 워크스페이스의 **data 탭**에서도 동일하게 가능합니다. 어느 쪽이든 데이터셋을 우클릭하면 메뉴에 **다운샘플** 항목이 보입니다. 같은 우클릭 메뉴에 Edit / Augment / Merge / Export / Delete 가 함께 들어 있어 다른 작업과 똑같은 진입점을 씁니다.

## 사용법

<Steps>
1. **데이터셋을 우클릭하고 다운샘플을 누릅니다.**

   다이얼로그 제목은 **데이터셋 다운샘플**입니다.

   <DocsImage
     src="/docs/datasets/downsample-01-dialog.png"
     alt="데이터셋 다운샘플 다이얼로그 — name / 유지 비율 / N프레임마다 1프레임 유지"
     status="missing"
   />

2. **새 데이터셋 이름과 비율을 입력합니다.**

   - **name** — 새로 생성할 데이터셋 이름. 기본값은 `<원본이름>_downsampled`.
   - **유지 비율 (keep)** — 한 블록에서 살릴 프레임 수. 기본값 `1`.
   - **N프레임마다 1프레임 유지 (every)** — 블록 크기. 기본값 `2`.

   예) `keep=1, every=2` → 2 프레임마다 1 프레임 유지 (절반으로 줄임).
   예) `keep=1, every=3` → 3 프레임마다 1 프레임 유지 (1/3).

   <Callout type="warning">
   `keep > 0`, `every > 0`, **`keep < every`** 조건을 모두 만족해야 합니다. 위반하면 백엔드가 400 오류(`Invalid downsample ratio`)를 돌려줍니다.
   </Callout>

3. **다운샘플 버튼을 누르면 비동기 작업이 시작됩니다.**

   토스트로 "다운샘플을 시작했습니다." 가 뜨고, 진행률은 socket 이벤트(`downsample_progress`)로 갱신됩니다. 완료되면 `downsample_complete` 이벤트가 전송되며 데이터셋 목록에 `_downsampled` 데이터셋이 새 행으로 추가됩니다.
</Steps>

## 그리퍼 transition 자동 보존

다운샘플은 기본적으로 **그리퍼(tool) transition을 보존**합니다. 그리퍼 open/close 동작은 보통 2~3 프레임 안에 끝나는데, 단순히 매 N 프레임마다 샘플링하면 phase에 따라 transition 전체가 빠질 수 있고, 결과적으로:

- 학습 시 같은 입력에 open / close 라벨이 섞여 라벨이 노이즈처럼 보임
- 추론 시 그리퍼가 진동하거나 결정을 못 내림

이를 막기 위해 데이터셋의 `info.json`에 `tool_qpos_indices`가 있으면 그리퍼 값이 크게 변하는 프레임은 sampling 위치와 상관없이 살아남고, 좌우 1프레임(`tool_pad=1`)도 함께 보존됩니다. 사용자는 별도 옵션을 만질 필요 없이 keep/every 비율만 정하면 됩니다.

<Callout type="tip">
다운샘플된 데이터셋도 [학습](/docs/training/dataset-select)에서 원본과 동일하게 선택할 수 있습니다. 같은 작업의 30Hz 원본과 10Hz 다운샘플 버전을 분리해 두면, 정책 hz에 맞는 쪽만 골라 학습할 수 있어 편리합니다.
</Callout>
