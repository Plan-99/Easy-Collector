수집한 데이터셋과 에피소드는 좌측 네비의 **데이터셋 탭**에서 한 곳에 모아 관리합니다.
워크스페이스의 **data 탭**에서도 같은 작업을 할 수 있어, 수집 직후 바로 정리하는 흐름에 편합니다.

<Callout type="info">
**리플레이는 어디로?** 에피소드를 실제 로봇으로 다시 흘려보내는 리플레이는 [데이터 수집 → 에피소드 리플레이](/docs/data-collection/replay)로 옮겼습니다. 수집 직후 결과 확인용이라 데이터 수집 섹션 안에 두는 편이 자연스럽기 때문입니다.
</Callout>

## 데이터셋을 분리해야 할 때

| 분리 전략 | 예시 |
|---------|------|
| **버전별** | `pick_v1` (초기 시연) / `pick_v2` (개선) |
| **시점별** | `pick_morning` / `pick_afternoon` (조명 차이) |
| **시연자별** | `pick_alice` / `pick_bob` (스타일 차이) |
| **train / eval** | `pick_train` / `pick_eval` (학습·평가 분리) |

## 가능한 작업

- [데이터셋 편집](/docs/datasets/edit) — 이름 변경, 병합, sensor/robot 매핑 재지정, 삭제 (데이터셋 단위)
- [일괄 변경 (Batch Edit)](/docs/datasets/batch-edit) — 증강·다운샘플·크롭·회전을 한 번에 적용해 새 데이터셋 생성 (데이터셋 단위)
- [데이터셋 증강](/docs/datasets/augment) · [데이터셋 다운샘플](/docs/datasets/downsample) — 일괄 변경 패널의 Augmentation / Downsample 섹션에서 적용. 각 기능별 상세 설명 페이지.
- [에피소드 편집](/docs/datasets/episode-edit) — 영상/센서 미리보기, trim, 자연어 지시 수정 (에피소드 단위)
- [데이터셋 불러오기 / 내보내기](/docs/datasets/import-export) — 아카이브로 백업하거나 다른 머신과 공유

<Callout type="info">
**우측 패널은 둘 중 하나만 뜹니다.** 좌측 트리에서 **데이터셋 폴더 라벨**을 클릭하면 우측에 [일괄 변경 패널](/docs/datasets/batch-edit)이, **에피소드 행**을 클릭하면 [에피소드 편집 패널](/docs/datasets/episode-edit)이 뜹니다. 폴더의 expand/collapse는 행 우측 chevron 으로만 동작합니다.
</Callout>

## 다음 단계

원하는 데이터셋이 준비되면 [학습](/docs/training/dataset-select)에서 한 번에 여러 데이터셋을 선택해 학습할 수 있습니다.

<Callout type="tip">
같은 작업의 여러 버전을 한 번에 학습하면 일반화에 도움이 됩니다. 예: `pick_v1`(20 ep) + `pick_v2`(15 ep)를 함께 선택해 35 ep 정책 학습.
</Callout>
