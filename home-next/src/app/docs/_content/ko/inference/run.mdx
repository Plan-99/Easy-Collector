<Steps>
1. **체크포인트 선택 후 MonitoringWindow에 inference 헤더가 나타납니다. Start Inference를 누르면 추론이 시작되고, 이상하면 Stop을 누릅니다.**

   - **Hz** — 추론 주기. 보통 30 Hz로 시작
   - **🏠 배지** — 추론 시작 시 Home Pose로 갈지 (파란색=ON, 회색=OFF)
   - **Start Inference** — 빨간색 play 버튼. 누르면 매 프레임마다 (이미지 + 관절 상태)를 받아 다음 액션을 추론하고 로봇에 보냅니다.

   <DocsImage
     src="/docs/inference/run-01-monitoring.png"
     alt="MonitoringWindow inference — Hz / Home 배지 / Start·Stop 버튼 + 추론 중 카메라 뷰"
   />

2. **상단의 Succeed 배지로 추론 품질을 모니터링합니다.**

   - **Succeed** — 0.7 이상이면 녹색. 학습된 성공 분포와 닮은 정도

3. **(선택) 추론 헤더 우측의 부가 기능을 활용합니다.**

   - **Vision Map 드롭다운** — ACT 정책이 영상의 어디를 보고 있는지 카메라 위에 실시간 히트맵으로 표시합니다. 자세한 사용법은 [비전 맵](/docs/inference/vision-map)을 참고하세요. (ACT 정책일 때만 보입니다.)
   - **노란 DAgger 버튼** — 정책이 stuck 상태일 때 그 자세 그대로 텔레옵으로 전환해 시연을 추가합니다. 자세한 흐름은 [DAgger](/docs/inference/dagger)를 참고하세요.
</Steps>

## reinference 주기

설정에 따라 정책이 한 번에 여러 step의 액션을 chunk로 출력합니다 (ACT의 chunk_size 등).
재추론(reinference) 주기는 정책 학습 시 설정한 chunk_size와 추론 Hz의 관계로 결정됩니다.

<Callout type="info">
추론 시간이 매 스텝마다 미세하게 달라져도 로봇이 부드럽게 움직이도록, 내부 보간 노드가 각 웨이포인트의 도착 시각을 기준으로 자동 스케줄링하고 출력단에 저역 통과 필터를 적용합니다. 사용자가 별도로 설정할 항목은 없습니다.
</Callout>

<Callout type="warning">
Succeed 점수가 낮게 유지되거나 동작이 학습 시연과 동떨어져 보이면 즉시 Stop을 누르고, 데이터를 더 모은 뒤 다시 시도하세요.
</Callout>
