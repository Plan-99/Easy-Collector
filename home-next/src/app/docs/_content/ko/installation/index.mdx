Easy Trainer는 Ubuntu 22.04 이상에서 동작하는 **Debian 패키지(.deb)** 형태로 배포됩니다.
설치는 4단계로 나뉩니다.

1. **시스템 요구사항 확인** — Docker, GPU(선택), 디스크 여유
2. **deb 패키지 설치** — 한 줄 명령
3. **런처 첫 실행** — Google 로그인(Device Flow)
4. **설치 마법사** — CPU/GPU와 학습 서버 옵션 선택, Docker 이미지 빌드

---

## 시스템 요구사항

| 항목 | 최소 | 권장 |
|------|------|------|
| OS | Ubuntu 22.04 LTS | Ubuntu 22.04 / 24.04 |
| 저장 공간 | 20 GB | 50 GB+ |
| RAM | 8 GB | 16 GB+ |
| GPU | (CPU만으로도 가능) | NVIDIA, VRAM 8 GB+ |

저장 공간은 Docker 이미지(약 12–15 GB) + 데이터셋 + 체크포인트가 모두 `/opt/easytrainer/`에 쌓이는 것을 감안한 권장값입니다.

## 사전 준비

deb 패키지의 `Recommends`에 docker가 있지만 자동 설치되지 않습니다. **Docker만 직접 설치**하고 나머지는 deb이 처리합니다.

### Docker 설치 확인

```bash
docker --version
docker compose version
```

<DocsImage
  src="/docs/installation/01-docker-version.png"
  alt="docker --version 명령 출력"
  caption="docker version 확인"
  status="missing"
/>

설치되어 있지 않다면 아래 명령으로 설치합니다.

```bash
sudo apt install -y curl
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$UBUNTU_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker "$USER" && newgrp docker
```

<Callout type="warning">
`docker` 그룹에 처음 추가됐다면 **로그아웃 후 다시 로그인**해야 권한이 적용됩니다. `newgrp docker`만으로는 그래픽 세션의 다른 프로세스에 권한이 전파되지 않습니다.
</Callout>

### NVIDIA GPU 환경 (선택)

GPU 가속을 사용하려면 **NVIDIA 드라이버 + NVIDIA Container Toolkit**이 필요합니다.

```bash
nvidia-smi
docker run --rm --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi
```

두 명령 모두 GPU 정보가 출력되면 OK입니다.

<DocsImage
  src="/docs/installation/02-nvidia-smi.png"
  alt="nvidia-smi와 docker --gpus 출력"
  status="missing"
/>

Container Toolkit이 없다면 다음과 같이 설치합니다.

```bash
sudo wget -qO /etc/apt/keyrings/nvidia-container-toolkit.asc \
  https://nvidia.github.io/libnvidia-container/gpgkey
echo "deb [signed-by=/etc/apt/keyrings/nvidia-container-toolkit.asc] \
https://nvidia.github.io/libnvidia-container/stable/deb/amd64 /" | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

<Callout type="info">
GPU가 없으면 설치 마법사에서 **CPU 버전**을 선택할 수 있습니다. 기능은 같지만 일부 정책(Diffusion, PI0)은 CPU에서 매우 느립니다.
</Callout>

### 디스크 여유 확인

```bash
df -h /
```

<DocsImage

## deb 패키지 설치

<Steps>
1. **deb 파일을 다운로드합니다.**

   <DocsImage
     src="/docs/installation/04-deb-download.png"
     alt="다운로드 페이지에서 easytrainer_*.deb 받기"
     status="missing"
   />

2. **apt로 설치합니다.**

   ```bash
   sudo apt install ./easytrainer_<version>_amd64.deb
   ```

   <DocsImage
     src="/docs/installation/05-apt-install.png"
     alt="apt install 출력"
     status="missing"
   />

   설치가 끝나면 다음 메시지가 표시됩니다.

   ```text
   Setting up easytrainer (<version>) ...
   User <username> added to docker group. Please log out/in to apply.
   Project placed at /opt/easytrainer/project (owner: <username>)
   ```

   - `/opt/easytrainer/project/` — 운영자 UI / 백엔드 / ROS 2 소스
   - `/opt/easytrainer/database/main.db` — SQLite DB (워크스페이스, 데이터셋 메타)
   - `/opt/easytrainer/datasets/` — 수집된 에피소드
   - `/opt/easytrainer/training_data/` — 학습 산출물
   - `/tmp/easytrainer/logs/` — 런타임 로그
</Steps>

## 런처 첫 실행

<Steps>
1. **런처를 실행합니다.**

   ```bash
   easytrainer-launcher
   ```

   또는 애플리케이션 메뉴에서 **Easy Trainer**를 실행해도 됩니다.

   <DocsImage
     src="/docs/installation/06-launcher-launch.png"
     alt="easytrainer-launcher 첫 실행"
     status="missing"
   />

   런처는 가장 먼저 다음 3가지를 점검합니다.

   - **데이터/프로젝트 경로 권한** (`/opt/easytrainer/`)
   - **Google 로그인 (디바이스 인증)**
   - **Docker 설치 여부**

   하나라도 실패하면 빨간 다이얼로그를 띄우고 종료합니다.

2. **Google 계정으로 로그인합니다.**

   런처가 자동으로 기본 브라우저를 열어 디바이스 등록 페이지로 이동시킵니다. 브라우저가 열리지 않으면 다이얼로그의 URL을 복사해서 직접 여세요.

   <DocsImage
     src="/docs/installation/07-login-dialog.png"
     alt="Easy Trainer 로그인 다이얼로그"
     status="missing"
   />

   <DocsImage
     src="/docs/installation/08-browser-oauth.png"
     alt="브라우저 OAuth 동의 화면"
     status="missing"
   />

   브라우저에서 Google 로그인 후 **이 기기 등록** 버튼을 누르면 런처가 자동으로 인증을 완료하고 토큰을 `~/.easytrainer/auth.json`에 저장합니다.

   <Callout type="info">
   사용자당 활성 기기는 **1개**입니다. 다른 PC에서 이미 등록된 상태라면
   `DEVICE_ALREADY_BOUND` 오류가 표시됩니다. 홈페이지의 `/dashboard`에서 기존
   기기를 해제한 뒤 다시 시도하세요.
   </Callout>

   <Callout type="warning">
   인증은 **10분 안에** 완료해야 합니다. 시간 초과되면 다이얼로그가 닫히고 처음부터 다시 시작해야 합니다.
   </Callout>
</Steps>

## 설치 마법사

처음 실행이면 5단계 설치 마법사가 자동으로 열립니다.

<Steps>
1. **소프트웨어 사용권 계약 (EULA)**

   <DocsImage
     src="/docs/installation/09-installer-eula.png"
     alt="설치 마법사 EULA 페이지"
     status="missing"
   />

   본문을 읽고 **동의함** 체크박스를 켠 뒤 **다음**.

2. **설치 준비 — 저장 공간 확인**

   <DocsImage
     src="/docs/installation/10-installer-disk.png"
     alt="저장 공간 확인 페이지"
     status="missing"
   />

   현재 여유 공간이 20 GB 이상이어야 진행됩니다. 부족하면 다음 버튼이 비활성화됩니다.

3. **CPU/GPU 모드와 학습 서버 옵션 선택**

   <DocsImage
     src="/docs/installation/11-installer-mode.png"
     alt="CPU/GPU 및 학습 서버 선택"
     status="missing"
   />

   **모드:**

   - **GPU 버전 (권장)** — NVIDIA Container Toolkit이 자동 구성됩니다. 학습·추론이 빠릅니다.
   - **CPU 버전** — GPU가 없을 때 선택. ACT는 느리게라도 동작하지만 Diffusion / PI0는 사실상 사용 불가.

   **학습 서버:**

   - **로컬 학습 서버 설치** — 같은 PC에서 학습까지 진행. 별도 GPU 서버가 없다면 이 옵션.
   - **원격 학습 서버 사용** — GPU가 좋은 다른 PC에 학습 서버만 별도로 띄울 때. URL은 나중에 [Train 페이지](/docs/training/server)에서 입력합니다.

   GPU 정보 박스에 검출된 NVIDIA GPU 목록과 VRAM 사용량이 표시됩니다.

   <Callout type="info">
   설치 후에도 런처의 🔥 [학습 서버 관리](/docs/training/server)에서 모드를 바꿀 수 있습니다. 일단 가장 단순한 "GPU + 로컬 학습 서버"로 시작하는 것을 권장합니다.
   </Callout>

4. **설치 진행**

   <DocsImage
     src="/docs/installation/12-installer-progress.png"
     alt="설치 진행률"
     status="missing"
   />

   다음 작업이 순차로 진행됩니다.

   - `[INSTALL] docker compose build --no-cache` — 약 15–40분
   - `[GPU] NVIDIA Container Toolkit 설치/구성` (GPU 모드일 때)
   - `[POST] Running DB migrations` — SQLite 스키마 생성
   - `[POST] Bringing up service` — backend / frontend / ros2 컨테이너 기동
   - `[MODULE] (i/N) <module_name> 다운로드 중` — 필수 모듈 자동 설치
   - `[TRAIN-SERVER] ...` — 학습 서버 다운로드/설치 (로컬 모드일 때)

   진행 시간은 환경에 따라 **20–60분**입니다.

5. **완료**

   <DocsImage
     src="/docs/installation/13-installer-done.png"
     alt="설치 완료"
     status="missing"
   />

   **Easy Trainer 실행하기** 체크박스를 그대로 두고 **완료**를 누르면 메인 런처가 뜹니다.
</Steps>

| 📋 로그 | Launcher / Frontend / Backend / ROS 2 로그 4창 표시 |
| 🧩 [모듈 관리](/docs/modules) | 로봇·센서·확장 모듈 설치/제거 |
| 🔥 [학습 서버](/docs/training/server) | 로컬 학습 서버 설치/실행/중지 |
| ✕ 종료 | EXIT / RESTART / STOP SERVICE |

다음으로 [빠른 시작](/docs/quickstart) 가이드를 따라 첫 데이터를 수집하고 학습까지 진행해 보세요.

## 자주 마주치는 문제

### Docker 권한 오류

```text
docker: Got permission denied while trying to connect to the Docker daemon socket
```

`docker` 그룹에 추가는 됐지만 세션에 적용되지 않은 상태입니다. **로그아웃 후 다시 로그인**하면 해결됩니다.

### GPU가 컨테이너에서 안 보임

호스트에서는 `nvidia-smi`가 잘 되는데 컨테이너에서는 안 된다면:

```bash
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

### `libnvidia-egl-wayland.so.1.1.x: no such file or directory`

GPU 컨테이너 실행 시 흔히 만나는 오류입니다. 호스트의 실제 버전을 찾아 심볼릭 링크를 만듭니다.

```bash
ls /usr/lib/x86_64-linux-gnu/libnvidia-egl-wayland.so.1.*
ACTUAL=$(ls /usr/lib/x86_64-linux-gnu/libnvidia-egl-wayland.so.1.*.* | head -1)
sudo ln -sf "$ACTUAL" /usr/lib/x86_64-linux-gnu/libnvidia-egl-wayland.so.1.1.9
sudo systemctl restart docker
```

마지막 인자(`1.1.9`)는 **에러 메시지에 나온 버전 번호**로 바꿔주세요.

### 5173 / 5000 포트 충돌

다른 프로세스가 같은 포트를 점유 중이면 컨테이너가 뜨지 않습니다.

```bash
sudo lsof -i :5173
sudo lsof -i :5000
```

해당 프로세스를 종료하거나 종료가 어렵다면 PC를 재부팅합니다.

### "DEVICE_ALREADY_BOUND" 오류

다른 PC에서 같은 Google 계정으로 이미 등록되어 있다는 뜻입니다. 홈페이지의 `/dashboard`로 가서 기존 기기를 **해제(unbind)**한 뒤 다시 시도하세요.

### 디스크가 꽉 차서 더 이상 학습되지 않음

데이터셋과 체크포인트는 모두 `/opt/easytrainer/` 아래에 쌓입니다.

```bash
du -sh /opt/easytrainer/*
```

오래된 체크포인트를 [추론 → Delete Checkpoint](/docs/inference/select-checkpoint)로 정리하거나, 사용하지 않는 데이터셋을 [데이터셋 관리 → Delete](/docs/datasets/edit)에서 삭제하세요.

## 재설치 / 제거

```bash
# 패키지만 제거 (데이터 보존)
sudo dpkg -r easytrainer

# 데이터까지 완전 삭제
bash scripts/clean_easytrainer.sh
```

<Callout type="danger">
`clean_easytrainer.sh`는 `/opt/easytrainer/` 전체와 사용자 설정(`~/.easytrainer/`)을 지웁니다. 수집한 데이터셋과 학습 결과가 모두 삭제되니 먼저 [내보내기](/docs/installation)로 백업하세요.
</Callout>
