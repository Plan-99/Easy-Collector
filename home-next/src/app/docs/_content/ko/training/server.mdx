학습 서버는 별도 Docker 서비스로 5100 포트에서 동작합니다.
[설치 마법사](/docs/installation)에서 "로컬 학습 서버 설치"를 선택했다면 이미 설치되어 있을 것입니다. 아니라면 런처에서 별도로 설치할 수 있습니다.

## 런처에서 설치·관리

<Steps>
1. **런처 floating bar의 🔥 학습 서버를 누릅니다.**

   <DocsImage
     src="/docs/training/server-01-launcher-btn.png"
     alt="런처 학습 서버 버튼"
     status="missing"
   />

2. **학습 서버 관리 다이얼로그가 열립니다.**

   GPU 정보, 설치 상태, 실행 상태, 포트가 한눈에 보입니다.

   <DocsImage
     src="/docs/training/server-02-dialog.png"
     alt="학습 서버 관리 다이얼로그"
     status="missing"
   />

3. **(미설치 상태에서) 설치 버튼을 누릅니다.**

   GitHub Releases에서 `training-server-v<버전>.tar.gz`를 다운로드하고 압축을 해제합니다.

   <DocsImage
     src="/docs/training/server-03-install.png"
     alt="설치 진행 로그"
     status="missing"
   />

4. **실행 버튼을 누르면 백엔드가 학습 서버 모드로 재생성됩니다.**

   상태가 **🟢 실행 중**으로 바뀌고 로그가 흐르면 정상.

   <DocsImage
     src="/docs/training/server-04-running.png"
     alt="실행 중 상태 + 로그"
     status="missing"
   />
</Steps>

## 운영자 UI에 연결

<Steps>
1. **Train 페이지의 Step 3에서 Training Server URL을 입력합니다.**

   - 같은 PC: `http://localhost:5100`
   - 별도 서버: `http://<server-ip>:5100`

   <DocsImage
     src="/docs/training/server-05-frontend-url.png"
     alt="Training Server URL 입력"
     status="missing"
   />

2. **URL 옆 버튼으로 헬스체크합니다.**

   체크 마크(positive) + **GPU Available** 배지가 보이면 정상.

   <DocsImage
     src="/docs/training/server-06-health-ok.png"
     alt="health check OK + GPU Available"
     status="missing"
   />
</Steps>

<Callout type="warning">
GPU 메모리가 부족하면 학습이 OOM으로 죽을 수 있습니다. nvidia-smi로 사용 가능한 VRAM을 확인하고, 필요하면 batch size를 줄이세요.
</Callout>
