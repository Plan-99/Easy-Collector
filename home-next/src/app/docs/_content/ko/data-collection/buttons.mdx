수집 중 화면 하단에는 색이 다른 3개 버튼이 나타납니다. 각각의 의미가 다르므로 구분해서 사용해야 합니다.

## SUCCESS (C)

작업이 잘 진행된 시점에 누릅니다. 단축키는 **C**.

<DocsImage
  src="/docs/data-collection/btn-01-success.png"
  alt="SUCCESS 버튼"
  status="missing"
/>

해당 프레임에 **성공 라벨**이 붙습니다. 학습 시 OOD/Succeed 점수 계산에 활용됩니다. 같은 에피소드 안에서 한 번만 누르면 됩니다 (이후 비활성화).

## DONE

한 에피소드를 정상 완료할 때 누릅니다.

<DocsImage
  src="/docs/data-collection/btn-02-done.png"
  alt="DONE 버튼"
  status="missing"
/>

데이터셋에 저장되고 다음 에피소드로 자동 진행됩니다. **이때까지의 모든 프레임이 학습 데이터에 들어갑니다.**

## STOP

에피소드를 **버립니다**.

<DocsImage
  src="/docs/data-collection/btn-03-stop.png"
  alt="STOP 버튼"
  status="missing"
/>

지금까지 녹화된 프레임이 모두 폐기되고 데이터셋에 저장되지 않습니다. 실수로 잘못 시연했거나 작업이 실패했을 때 사용합니다.

<Callout type="tip">
"실패도 학습에 좋다"는 통념이 있지만, **명백히 잘못된 동작**은 STOP으로 빼는 게 정책 수렴에 도움이 됩니다. 단순히 느리거나 거칠었던 시연은 그대로 DONE으로 살리세요.
</Callout>
