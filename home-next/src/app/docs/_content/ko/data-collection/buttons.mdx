수집 중 화면 하단에는 색이 다른 4개 버튼이 나타납니다. 각각의 의미가 다르므로 구분해서 사용해야 합니다. 키보드 단축키는 **1 / 2 / 3 / 4**가 그대로 매핑되어 있습니다.

## 완료 (1)

작업이 잘 진행된 시점에 누릅니다. 단축키는 **1**.

해당 프레임에 **성공 라벨**이 붙고 버튼이 노란색으로 강조됩니다. 학습 시 Succeed 점수 계산에 활용됩니다. 같은 에피소드 안에서 한 번만 누르면 됩니다 (이후 비활성화). 이 라벨이 찍힌 체크포인트만 [플래너 Checkpoint 블록의 "Done 신호까지 실행"](/docs/planner/blocks-and-run#checkpoint-블록)을 사용할 수 있습니다.

## 종료 (2)

한 에피소드를 정상 완료할 때 누릅니다. 단축키는 **2**.

지금까지 녹화된 프레임이 데이터셋에 저장되고 자동으로 다음 에피소드 수집이 이어집니다. **이때까지의 모든 프레임이 학습 데이터에 들어갑니다.**

## 버리기 (3)

현재 에피소드를 **데이터셋에 저장하지 않고 폐기**한 뒤 곧바로 다음 에피소드로 넘어갑니다. 단축키는 **3**.

실수로 잘못 시연했거나 작업이 실패했을 때 사용합니다. **데이터 수집 자체는 계속 이어지므로**, 한 번 망쳤다고 처음부터 다시 세팅할 필요 없이 즉시 다음 시연으로 복귀할 수 있습니다.

## 중지 (4)

데이터 수집 세션 자체를 끝냅니다. 단축키는 **4**.

현재 녹화 중인 에피소드는 저장되지 않고 사라지며, 다음 에피소드도 자동으로 시작되지 않습니다. 충분히 모았거나 잠시 작업을 멈춰야 할 때 누릅니다. 다시 모으려면 REC 버튼으로 새로 시작해야 합니다.

<Callout type="tip">
"실패도 학습에 좋다"는 통념이 있지만, **명백히 잘못된 동작**은 버리기(3)로 빼는 게 정책 수렴에 도움이 됩니다. 단순히 느리거나 거칠었던 시연은 그대로 종료(2)로 살리세요.
</Callout>

<Callout type="info">
폼 입력 중(예: 자연어 지시 텍스트박스에 포커스가 있을 때)에는 1/2/3/4 단축키가 동작하지 않습니다. 화면 빈 영역을 한 번 클릭한 뒤 단축키를 사용하세요.
</Callout>
