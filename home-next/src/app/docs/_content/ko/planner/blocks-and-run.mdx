플래너의 우측 상단 **Plans 타임라인**은 그룹마다 한 줄씩 가로로 늘어선 카드 트랙입니다. 카드 한 장이 블록 하나이고, 왼쪽에서 오른쪽 순서로 실행됩니다. 그룹은 서로 로봇을 공유하지 않으므로 **여러 그룹을 동시에** 돌릴 수 있습니다. 각 블록의 결과(완료 · 중지 · 오류)는 실행 후 카드 우측 상단에 아이콘으로 남습니다.

## 블록 추가하기

<Steps>
1. **추가하려는 그룹 트랙의 오른쪽 끝, + 블록 추가 버튼을 누릅니다.**

   같은 플래너 안에 그룹이 여러 개라면 각 그룹마다 별도의 + 버튼이 있습니다. 블록은 누른 그룹에 들어갑니다.

   <DocsImage
     src="/docs/planner/blocks-01-new-button.png"
     alt="새 블록 버튼 (그룹 트랙 우측 끝)"
   />

2. **블록 종류를 선택합니다.**

   다섯 가지 중 하나를 고를 수 있습니다. 선택한 종류에 따라 다이얼로그 아래쪽 입력 항목이 바뀝니다.

   - **Joint Position** — 워크스페이스 로봇을 저장된 관절 자세로 이동
   - **Checkpoint** — 학습된 체크포인트로 추론 실행 (시간 종료 또는 Done까지)
   - **Query Pose** — 외부 ROS 서비스에서 받아온 좌표로 로봇 이동 (관절 / 엔드 이펙터)
   - **Time Sleep** — 지정한 초만큼 대기
   - **Sync** — 다른 그룹과 만나는 동기화 지점

   <DocsImage
     src="/docs/planner/blocks-02-type-select.png"
     alt="블록 종류 선택"
   />

3. **종류별 세부 항목을 채우고 다이얼로그 우측 하단의 저장을 누릅니다.**

   종류와 무관하게 공통으로 입력하는 항목은 다음 한 가지뿐입니다.

   - **이름** *(선택)* — 비워 두면 종류와 워크스페이스/체크포인트 이름으로 자동 생성됩니다. 예: `이동 (워크스페이스 A)`, `pick_v3 실행`, `2초 대기`, `Sync start`.

   나머지 입력 항목은 선택한 블록 종류에 따라 다릅니다. 각 종류별 입력 항목은 아래 [Joint Position 블록](#joint-position-블록), [Checkpoint 블록](#checkpoint-블록), [Time Sleep 블록](#time-sleep-블록), [Sync 블록](#sync-블록) 절을 참고하세요.
</Steps>

## Joint Position 블록

지정 워크스페이스의 로봇들을 저장된 관절 자세로 이동시킵니다.

<Steps>
1. **워크스페이스를 선택합니다.**

   해당 워크스페이스의 어셈블리에 포함된 모든 로봇의 자세를 한 번에 저장합니다. 선택할 수 있는 워크스페이스는 **이 블록이 속한 그룹**의 워크스페이스로 자동으로 좁혀집니다.

   <DocsImage
     src="/docs/planner/blocks-joint-01-workspace.png"
     alt="Joint Position — 워크스페이스 선택"
   />

2. **로봇 펜던트로 원하는 자세를 만듭니다.**

   워크스페이스에 포함된 각 로봇별로 펜던트가 표시됩니다. 로봇이 켜져 있어야 펜던트가 보이며, 꺼져 있으면 안내 문구가 대신 표시됩니다.

   <DocsImage
     src="/docs/planner/blocks-joint-02-pendant.png"
     alt="블록 다이얼로그 안의 로봇 펜던트"
   />

3. **현재 자세 적용 버튼으로 자세를 저장합니다.**

   현재 로봇의 관절값이 저장된 자세 카드(상단 강조 박스)에 표시됩니다. 저장된 자세는 **이동** 버튼으로 미리 검증해 볼 수 있습니다.

   <DocsImage
     src="/docs/planner/blocks-joint-03-apply.png"
     alt="현재 자세 적용 / 이동 버튼"
   />
</Steps>

## Checkpoint 블록

선택한 체크포인트로 워크스페이스에서 실시간 추론을 실행합니다. 끝나는 조건은 **시간**이거나 **Done 신호**입니다.

<Steps>
1. **워크스페이스와 체크포인트를 선택합니다.**

   체크포인트 목록은 선택한 워크스페이스에서 학습된 항목으로 자동으로 좁혀집니다.

2. **추론 종료 방식과 시간(또는 Done 임계값)을 정합니다.**

   - **Done 신호까지 실행** *(토글)* — 켜면 정책의 `done` 점수가 임계값을 넘는 순간 즉시 다음 블록으로 넘어갑니다. 끄면 지정 시간만큼만 실행합니다.
   - **Done 임계값** — Done 토글이 켜졌을 때만 보입니다. 0과 1 사이 값 (기본 0.5).
   - **시간 (초)** — Done 토글이 꺼졌을 때 사용. 이 블록이 실행될 시간.
   - **추론 Hz** — 초당 정책을 호출하는 횟수.
   - **re_inference_steps** — 한 번 추론한 액션을 몇 step 동안 재사용할지.
   - **temporal_ensemble_coeff** — `re_inference_steps=1`일 때만 표시. 시간 평활 계수.

   <DocsImage
     src="/docs/planner/blocks-checkpoint.png"
     alt="Checkpoint 블록 — 워크스페이스/체크포인트, Done 토글, 파라미터"
   />
</Steps>

<Callout type="warning">
**Done 신호까지 실행**은 체크포인트가 [학습 시 has_succeed 옵션으로 학습](/docs/training)되어 있어야 동작합니다. has_succeed 없이 학습된 체크포인트는 토글을 켤 수 없고 안내 문구가 표시됩니다.
</Callout>

<Callout type="info">
체크포인트 블록 실행 중에도 [추론](/docs/inference)과 동일하게 정책이 자율적으로 로봇을 움직입니다. 안전한 환경에서 시작하고, 이상 동작이 보이면 곧바로 **중지**를 누르세요.
</Callout>

## Query Pose 블록

외부 ROS2 서비스를 호출해 받은 좌표로 로봇을 이동시킵니다. 비전 인식 노드처럼 매 실행마다 위치가 달라지는 목표(예: 인식된 박스 중심)에 로봇을 보낼 때 사용합니다. 서비스 응답은 어셈블리 슬롯 이름을 키로 하는 JSON 객체로 와야 합니다.

<Steps>
1. **워크스페이스를 선택합니다.**

   같은 그룹 안의 워크스페이스 중 하나를 고릅니다. 워크스페이스가 하나뿐이면 자동으로 그 값이 들어가고 셀렉트는 표시되지 않습니다.

2. **포즈 타입을 선택합니다.**

   서비스가 응답으로 돌려주는 좌표의 종류입니다.

   - **관절 좌표** — 슬롯별 관절 각도 배열 (예: `[0.1, -0.5, 0.7, ...]`)
   - **엔드 이펙터 좌표** — 슬롯별 EE 위치/자세. 백엔드가 IK로 풀어 관절 값으로 변환합니다 (IK가 활성화된 단일 EE 로봇만 지원).

3. **호출할 서비스 이름을 입력합니다.**

   형식은 `std_srvs/srv/Trigger`입니다. 예: `/my_target`. 응답 메시지의 `message` 필드에 JSON 문자열이 담겨 있어야 합니다.

4. **외부 노드 예시 보기 버튼으로 서버 예시 코드를 확인합니다.**

   다이얼로그 안에 `std_srvs` 만으로 동작하는 ROS2 노드 예시 코드가 표시됩니다. 클립보드로 복사한 뒤 호스트에서 `python3 my_target_server.py`로 실행하면 위 서비스 이름으로 응답을 제공합니다 (별도 colcon build 필요 없음).

5. **이동 시간(초)을 입력합니다.**

   서비스에서 받아온 목표 자세까지 보간 이동하는 데 걸리는 시간입니다. 짧을수록 빠르게 움직이지만 속도/저크가 커집니다.

6. **서비스 호출 + 로봇 이동 테스트 버튼으로 실제 동작을 검증합니다.**

   서비스를 한 번 호출해서 받아온 좌표로 로봇을 이동시키고, 응답 JSON을 다이얼로그 안에 보여 줍니다. 좌표 형식이 맞지 않거나 IK가 풀리지 않으면 빨간 오류 메시지가 표시되므로, 플랜에 추가하기 전에 미리 검증할 수 있습니다.
</Steps>

<Callout type="warning">
외부 ROS2 서비스 노드는 **사용자가 직접 띄워야** 합니다. EasyTrainer가 자동으로 실행하지 않습니다. 서비스가 떠 있지 않으면 블록 실행 시 호출 실패로 끝납니다.
</Callout>

<Callout type="info">
**엔드 이펙터 좌표 모드**는 워크스페이스의 로봇에 IK가 활성화되어 있어야 하고, 어셈블리당 EE가 하나여야 합니다 (in-container 플래너와 동일 제약). 듀얼 EE나 IK 미설정 로봇이면 백엔드가 시작 검증에서 실패시킵니다.
</Callout>

## Sync 블록

다른 그룹과 시점을 맞추는 **동기화 지점**입니다. 같은 `sync_id`를 가진 Sync 블록이 다른 그룹에 있으면, 양쪽이 그 지점에 도달할 때까지 서로 기다린 다음 함께 다음 블록으로 넘어갑니다.

<Steps>
1. **동기화 ID를 입력합니다.**

   짧은 문자열이면 무엇이든 됩니다 (예: `pick_ready`, `step1`, `A`). 짝을 맞추고 싶은 다른 그룹의 Sync 블록과 **정확히 같은 문자열**이어야 합니다.

2. **다른 그룹의 같은 위치에도 같은 ID로 Sync 블록을 둡니다.**

   예: 왼팔 그룹과 오른팔 그룹이 모두 "물체 잡기" 직후에 같이 멈춰서 만나야 한다면, 두 그룹의 그 위치에 각각 `sync_id=pick_ready`로 Sync 블록을 추가합니다.
</Steps>

<Callout type="info">
- **그룹이 하나뿐이거나** 같은 ID가 한 그룹에만 있으면 Sync 블록은 **no-op**으로 즉시 통과합니다 — 안전하게 두어도 됩니다.
- 어떤 그룹이 오류로 끝나거나 사용자가 그룹 하나만 멈춰도, 그 그룹은 자동으로 배리어에서 빠지므로 다른 그룹이 무한히 기다리는 일은 없습니다.
- Sync 블록은 로봇을 건드리지 않습니다. 정말로 대기만 합니다.
</Callout>

## 블록 관리

추가된 블록은 그룹 트랙의 가로 카드로 표시됩니다. 각 카드에서 다음 동작을 할 수 있습니다.

- **드래그** — 카드를 끌어 같은 트랙 안에서 좌우로 옮기면 실행 순서가 바뀝니다. (실행 중에는 비활성화)
- **우클릭** — 컨텍스트 메뉴로 **상세 보기**, **블록 편집**, **블록 복제**, **블록 삭제** 사용
- **배지** — Time Sleep / Checkpoint / Query Pose 블록은 `Ns` 형태로 시간이, Done 토글이 켜진 Checkpoint는 **Done까지** 배지가, Sync 블록은 **Sync** 배지가, Query Pose 블록은 호출할 서비스 이름이 카드 위에 붙습니다.

<DocsImage
  src="/docs/planner/blocks-list-01-row.png"
  alt="그룹 트랙의 블록 카드와 우클릭 메뉴"
/>

카드 좌측의 색 막대는 종류를 색으로 구분합니다 (관절 이동: 파랑, 체크포인트: 보라, Query Pose: 남보라, 시간 대기: 주황, Sync: 청록).

## 플랜 실행

플래너는 두 가지 실행 방식을 지원합니다.

- **전체 실행** *(타임라인 상단 우측)* — 모든 그룹을 **병렬**로 동시에 시작합니다. 그룹 안에서는 블록이 순차로 실행되고, Sync 블록을 만나면 짝이 도달할 때까지 기다립니다.
- **이 그룹만 실행** *(각 그룹 트랙 좌측)* — 한 그룹만 단독으로 실행합니다. 다른 그룹의 Sync 블록은 짝이 없으므로 no-op으로 넘어갑니다.

<Steps>
1. **실행 방식을 골라 버튼을 누릅니다.**

   서버는 시작 직전에 다음을 검증합니다. 하나라도 실패하면 알림으로 원인이 표시되고 실행이 시작되지 않습니다.

   - 사용 중인 워크스페이스가 모두 존재하는지
   - 실행할 그룹의 블록이 참조하는 로봇이 모두 켜져 있는지
   - 체크포인트 블록의 체크포인트가 존재하고 정책이 연결되어 있는지
   - **Done 신호까지 실행**이 켜진 체크포인트가 `has_succeed`로 학습되어 있는지
   - Time Sleep 블록의 시간이 0보다 큰지
   - Query Pose 블록의 서비스 이름이 비어 있지 않고 (EE 모드일 때) 워크스페이스 로봇이 IK 가능한 단일 EE 로봇인지
   - Sync 블록의 동기화 ID가 비어 있지 않은지

2. **현재 실행 중인 블록 카드가 파란 테두리와 스피너로 강조됩니다.**

   상단 상태 텍스트에는 그룹별 진행 정보가 표시됩니다. 다른 그룹의 Sync 블록을 기다리는 중에는 **대기 중** 상태로 보입니다.

3. **하단 모니터링 윈도우에서 센서/로봇 상태를 함께 확인합니다.**

   체크포인트 블록 실행 시 정책이 보내는 액션이 실시간으로 로봇에 반영됩니다. 모니터링 패널이 타임라인 아래로 배치되어 페이지를 자연 스크롤하면서 보면 됩니다.

4. **각 블록이 끝나면 카드 우측 상단에 결과 아이콘이 남습니다.**

   - check_circle (녹색) — 정상 완료
   - cancel (주황) — 사용자 중지
   - error (빨강) — 오류로 종료. 오류 내용은 알림으로 표시됩니다.

5. **이상하거나 위험해 보이면 중지 버튼을 누릅니다.**

   **전체 실행** 상태에서 누르면 모든 그룹이 즉시 중지됩니다. Sync 블록에서 기다리던 그룹도 함께 풀려납니다.

</Steps>

## 실행 중 제약

- 실행 중에는 새 블록 추가, 블록 편집/삭제/복제, 카드 드래그, 우클릭 메뉴 모두 비활성화됩니다.
- 페이지를 새로 열어도 서버에서 실행 상태를 다시 받아오므로, 다른 탭/창에서도 진행 중임이 그대로 표시됩니다.

<Callout type="danger">
플래너 실행 중에는 정책이 자율적으로 로봇을 움직이는 구간(Checkpoint 블록)과 저장된 자세로 곧장 이동하는 구간(Joint Position 블록)이 섞여 있고, 전체 실행 시에는 **여러 로봇이 동시에 움직입니다**. 사람이 항상 비상 정지 가능한 거리에서 감시하고, 실행 환경에 사람이나 깨지기 쉬운 물체가 없는지 확인한 뒤 시작하세요.
</Callout>

## 잘 안 될 때

| 증상 | 가능한 원인 | 대처 |
|------|---------|------|
| 시작 직후 알림이 뜨고 실행되지 않음 | 사전 검증 실패 (로봇 꺼짐 / 체크포인트 누락 등) | 알림 메시지의 블록 위치로 이동해 항목을 채웁니다. |
| Joint Position 블록에서 로봇이 움직이지 않음 | 저장된 자세가 비어 있거나 펜던트로 저장하지 않음 | 블록 편집 → **현재 자세 적용**으로 다시 저장합니다. |
| Done 토글이 꺼져 있고 켜지지 않음 | 체크포인트가 `has_succeed` 없이 학습됨 | 학습 설정에 `has_succeed`를 켜고 [다시 학습](/docs/training)하거나, 토글을 끄고 시간으로 종료합니다. |
| Sync 블록에서 영원히 기다림 | 다른 그룹의 짝 Sync 블록이 누락 / ID 불일치 | 양쪽 그룹의 `sync_id`가 정확히 같은지 확인하거나, 짝이 없으면 Sync 블록을 삭제합니다. |
| Checkpoint 블록 도중 동작이 떨림 | 추론 Hz가 학습 시 데이터 수집 Hz와 다름 | 추론 Hz를 학습 데이터 수집 Hz와 맞춥니다. |
| 도중에 빨간 error 아이콘으로 끝남 | 정책 실행 중 예외 / 토픽 끊김 등 | 알림의 오류 메시지를 확인하고 [추론](/docs/inference) 문제 해결 표를 함께 참고합니다. |
