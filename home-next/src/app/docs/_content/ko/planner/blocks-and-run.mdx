플래너의 가운데 패널에서 실행할 블록을 추가하고 순서를 정한 뒤, **전체 실행** 버튼으로 플랜을 한 번에 시작합니다. 위에서 아래로 순차 실행되며, 각 블록의 결과(완료 · 중지 · 오류)는 실행 후 행 우측에 아이콘으로 표시됩니다.

## 블록 추가하기

<Steps>
1. **가운데 패널 상단의 새 블록 버튼을 누릅니다.**

   블록 생성 다이얼로그가 열립니다.

   <DocsImage
     src="/docs/planner/blocks-01-new-button.png"
     alt="새 블록 버튼"
   />

2. **블록 종류를 선택합니다.**

   세 가지 중 하나를 고를 수 있습니다. 선택한 종류에 따라 다이얼로그 아래쪽 입력 항목이 바뀝니다.

   - **Joint Position** — 워크스페이스 로봇을 저장된 관절 자세로 이동
   - **Checkpoint** — 학습된 체크포인트로 일정 시간 추론 실행
   - **Time Sleep** — 지정한 초만큼 대기

   <DocsImage
     src="/docs/planner/blocks-02-type-select.png"
     alt="블록 종류 선택"
   />

3. **종류별 세부 항목을 채우고 다이얼로그 우측 하단의 저장을 누릅니다.**

   종류와 무관하게 공통으로 입력하는 항목은 다음 한 가지뿐입니다.

   - **이름** *(선택)* — 비워 두면 종류와 워크스페이스/체크포인트 이름으로 자동 생성됩니다. 예: `이동 (워크스페이스 A)`, `pick_v3 실행`, `2초 대기`.

   나머지 입력 항목은 선택한 블록 종류에 따라 다릅니다. 각 종류별 입력 항목은 아래 [Joint Position 블록](#joint-position-블록), [Checkpoint 블록](#checkpoint-블록), [Time Sleep 블록](#time-sleep-블록) 절을 참고하세요.
</Steps>

## Joint Position 블록

지정 워크스페이스의 로봇들을 저장된 관절 자세로 이동시킵니다.

<Steps>
1. **워크스페이스를 선택합니다.**

   해당 워크스페이스의 어셈블리에 포함된 모든 로봇의 자세를 한 번에 저장합니다.

   <DocsImage
     src="/docs/planner/blocks-joint-01-workspace.png"
     alt="Joint Position — 워크스페이스 선택"
   />

2. **로봇 펜던트로 원하는 자세를 만듭니다.**

   워크스페이스에 포함된 각 로봇별로 펜던트가 표시됩니다. 로봇이 켜져 있어야 펜던트가 보이며, 꺼져 있으면 안내 문구가 대신 표시됩니다.

   <DocsImage
     src="/docs/planner/blocks-joint-02-pendant.png"
     alt="블록 다이얼로그 안의 로봇 펜던트"
   />

3. **현재 자세 적용 버튼으로 자세를 저장합니다.**

   현재 로봇의 관절값이 저장된 자세 카드(상단 강조 박스)에 표시됩니다. 저장된 자세는 **이동** 버튼으로 미리 검증해 볼 수 있습니다.

   <DocsImage
     src="/docs/planner/blocks-joint-03-apply.png"
     alt="현재 자세 적용 / 이동 버튼"
   />
</Steps>

## Checkpoint 블록

선택한 체크포인트로 워크스페이스에서 실시간 추론을 일정 시간 동안 실행합니다.

<Steps>
1. **워크스페이스와 체크포인트를 선택합니다.**

   체크포인트 목록은 선택한 워크스페이스에서 학습된 항목으로 자동으로 좁혀집니다.

2. **시간(초)과 추론 Hz를 입력합니다.**

   - **시간 (초)** — 이 블록이 실행될 시간
   - **추론 Hz** — 초당 정책을 호출하는 횟수
   - **re_inference_steps** — 한 번 추론한 액션을 몇 step 동안 재사용할지
   - **temporal_ensemble_coeff** — `re_inference_steps=1`일 때만 표시. 시간 평활 계수

   <DocsImage
     src="/docs/planner/blocks-checkpoint.png"
     alt="Checkpoint 블록 — 워크스페이스/체크포인트 선택과 파라미터"
   />
</Steps>

<Callout type="info">
체크포인트 블록 실행 중에도 [추론](/docs/inference)과 동일하게 정책이 자율적으로 로봇을 움직입니다. 안전한 환경에서 시작하고, 이상 동작이 보이면 곧바로 **중지**를 누르세요.
</Callout>

## Time Sleep 블록

다음 블록까지 지정한 시간만큼 대기합니다. 정책이 끝난 직후 사용자가 결과를 확인하거나 환경을 정리할 시간을 주고 싶을 때 유용합니다.

<Steps>
1. **시간(초)을 입력하고 저장합니다.**

   0보다 큰 숫자여야 합니다.

   <DocsImage
     src="/docs/planner/blocks-sleep-01-duration.png"
     alt="Time Sleep 블록 시간 입력"
   />
</Steps>

## 블록 관리

추가된 블록은 가운데 패널 목록에 행으로 표시됩니다. 각 행에서 다음 동작을 할 수 있습니다.

- **드래그** — 행을 끌어 위/아래로 옮기면 실행 순서가 바뀝니다. (실행 중에는 비활성화)
- **우클릭** — 컨텍스트 메뉴로 **상세 보기**, **블록 편집**, **블록 복제**, **블록 삭제** 사용
- **타이머 배지** — Time Sleep과 Checkpoint 블록은 우측에 `Ns` 형태로 시간이 표시됩니다.

<DocsImage
  src="/docs/planner/blocks-list-01-row.png"
  alt="블록 목록 행과 우클릭 메뉴"
/>

블록 좌측의 색 막대는 종류를 색으로 구분합니다 (관절 이동: 파랑, 체크포인트: 보라, 시간 대기: 주황).

## 플랜 실행

블록 구성이 끝났다면 가운데 패널 상단의 **전체 실행** 버튼으로 플랜을 시작합니다. 한 블록이 끝나면 다음 블록이 이어서 실행됩니다.

<Steps>
1. **전체 실행 버튼(녹색 play 아이콘)을 누릅니다.**

   서버는 시작 직전에 다음을 검증합니다. 하나라도 실패하면 알림으로 원인이 표시되고 실행이 시작되지 않습니다.

   - 사용 중인 워크스페이스가 모두 존재하는지
   - 각 블록의 워크스페이스 로봇이 켜져 있는지
   - 체크포인트 블록의 체크포인트가 존재하고 정책이 연결되어 있는지
   - Time Sleep 블록의 시간이 0보다 큰지

2. **현재 실행 중인 블록이 파란 테두리와 스피너로 강조됩니다.**

   상단 상태 텍스트에는 진행 정보(`실행 중 (current/total) — 블록 이름`)가 표시됩니다.

3. **우측 모니터링 윈도우에서 센서/로봇 상태를 함께 확인합니다.**

   체크포인트 블록 실행 시 정책이 보내는 액션이 실시간으로 로봇에 반영됩니다.

4. **각 블록이 끝나면 행 우측에 결과 아이콘이 남습니다.**

   - check_circle (녹색) — 정상 완료
   - cancel (주황) — 사용자 중지
   - error (빨강) — 오류로 종료. 오류 내용은 알림으로 표시됩니다.

5. **이상하거나 위험해 보이면 중지 버튼을 누릅니다.**

   현재 실행 중인 블록이 즉시 종료되고, 이후 블록은 실행되지 않습니다.

   <DocsImage
     src="/docs/planner/run.png"
     alt="플랜 실행 중 — 좌측 블록 진행 상태(완료 체크 / 실행 중 스피너 / 중지 버튼)와 우측 모니터링 윈도우(센서·로봇 상태)"
   />
</Steps>

## 실행 중 제약

- 실행 중에는 새 블록 추가, 블록 편집/삭제/복제, 드래그 정렬, 우클릭 메뉴 모두 비활성화됩니다.
- 페이지를 새로 열어도 서버에서 실행 상태를 다시 받아오므로, 다른 탭/창에서도 진행 중임이 그대로 표시됩니다.

<Callout type="danger">
플래너 실행 중에는 정책이 자율적으로 로봇을 움직이는 구간(Checkpoint 블록)과 저장된 자세로 곧장 이동하는 구간(Joint Position 블록)이 섞여 있습니다. 사람이 항상 비상 정지 가능한 거리에서 감시하고, 실행 환경에 사람이나 깨지기 쉬운 물체가 없는지 확인한 뒤 시작하세요.
</Callout>

## 잘 안 될 때

| 증상 | 가능한 원인 | 대처 |
|------|---------|------|
| 시작 직후 알림이 뜨고 실행되지 않음 | 사전 검증 실패 (로봇 꺼짐 / 체크포인트 누락 등) | 알림 메시지의 블록 번호로 이동해 항목을 채웁니다. |
| Joint Position 블록에서 로봇이 움직이지 않음 | 저장된 자세가 비어 있거나 펜던트로 저장하지 않음 | 블록 편집 → **현재 자세 적용**으로 다시 저장합니다. |
| Checkpoint 블록 도중 동작이 떨림 | 추론 Hz가 학습 시 데이터 수집 Hz와 다름 | 추론 Hz를 학습 데이터 수집 Hz와 맞춥니다. |
| 도중에 빨간 error 아이콘으로 끝남 | 정책 실행 중 예외 / 토픽 끊김 등 | 알림의 오류 메시지를 확인하고 [추론](/docs/inference) 문제 해결 표를 함께 참고합니다. |
