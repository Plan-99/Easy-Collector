플래너는 좌측 메뉴의 **Planner** 페이지에서 사용합니다. 학습된 정책 한 개만으로 끝낼 수 없는 긴 작업을 여러 **블록**(관절 이동 · 체크포인트 실행 · 시간 대기 · 동기화)을 순서대로 엮어 자동으로 실행하고, 서로 로봇을 공유하지 않는 워크스페이스끼리는 **병렬**로 실행합니다.

## 사전 조건

플래너를 사용하려면 다음이 미리 준비되어 있어야 합니다.

- 하나 이상의 [워크스페이스](/docs/workspace)
- 워크스페이스에 연결된 [어셈블리와 로봇](/docs/assembly)
- 체크포인트 블록을 사용한다면 [학습이 끝난 체크포인트](/docs/training)

<Callout type="info">
플래너 실행은 워크스페이스의 로봇이 모두 켜진 상태에서만 시작할 수 있습니다. 시작 전에 좌측 워크스페이스 패널에서 각 로봇의 상태가 **켜짐**인지 확인하세요.
</Callout>

## 그룹 — 같은 로봇을 공유하는 워크스페이스 묶음

플래너에 워크스페이스를 추가하면 백엔드가 어셈블리에 포함된 로봇 ID를 비교해서 **그룹**을 자동으로 계산합니다.

- **로봇을 하나라도 공유하는 워크스페이스끼리는 같은 그룹**으로 묶입니다. (예: 두 워크스페이스가 모두 `right_arm`을 쓰면 같은 그룹)
- **공유 로봇이 없으면 서로 다른 그룹**이 됩니다. (예: 한쪽은 왼팔만, 한쪽은 오른팔만 쓰는 듀얼 셋업)
- 그룹은 워크스페이스를 추가/제거할 때마다 자동으로 다시 계산되며, 기존 블록은 가능한 한 원래 그룹에 그대로 남습니다.

같은 그룹 안의 블록들은 **위에서 아래로 순차 실행**되고, 서로 다른 그룹은 **동시에 병렬 실행**됩니다. 그룹은 같은 로봇을 절대 공유하지 않으므로 병렬로 돌아도 안전합니다.

## 페이지 구성

플래너 페이지는 좌측 패널 + 우측 타임라인 + 하단 모니터링으로 구성됩니다.

- **좌측 패널** — 이 플래너가 사용할 워크스페이스 목록. 각 워크스페이스 카드 안에 센서/로봇 토글이 있습니다.
- **우측 상단 — Plans 가로 타임라인** — 그룹마다 한 줄씩 가로로 카드가 늘어섭니다. 카드 한 장이 블록 하나입니다. 좌측에는 그룹 이름과 **이 그룹만 실행** 버튼, 우측 끝에는 **+ 블록 추가** 버튼이 있고, 그 위에 **전체 실행** / **중지** 버튼이 있습니다.
- **우측 하단 — 모니터링 윈도우** — 추가된 모든 워크스페이스의 센서 영상과 로봇 상태를 실시간으로 보여줍니다. 페이지는 자연 스크롤됩니다.

## 블록 종류

| 종류 | 색 | 의미 |
|------|----|------|
| **Joint Position** | 파랑 | 지정 워크스페이스의 로봇을 저장된 관절 자세로 이동시킵니다. |
| **Checkpoint** | 보라 | 지정 체크포인트로 실시간 추론을 실행합니다. 시간으로 끝내거나, **Done 신호까지** 실행할 수 있습니다. |
| **Query Pose** | 남보라 | 외부 ROS2 서비스(`std_srvs/srv/Trigger`)를 호출해 받아온 좌표로 로봇을 이동시킵니다. 관절 또는 엔드 이펙터 좌표를 지원합니다. |
| **Time Sleep** | 주황 | 지정한 초만큼 대기합니다. |
| **Sync** | 청록 | 다른 그룹과 만나는 **동기화 지점**. 같은 `sync_id`를 가진 Sync 블록이 다른 그룹에 있으면, 양쪽이 그 지점에 도달할 때까지 서로 기다립니다. |

<Callout type="info">
**Sync 블록이 필요한 이유** — 그룹은 기본적으로 독립적으로 흘러갑니다. 하지만 "왼팔이 물체를 잡은 다음에야 오른팔이 컵을 집어야 한다"처럼 그룹끼리 시점을 맞춰야 할 때가 있습니다. 두 그룹의 같은 위치에 같은 `sync_id`로 Sync 블록을 두면 그 지점에서 둘이 만난 뒤에 다음 블록으로 진행합니다. 워크스페이스가 하나뿐이거나, 같은 ID가 한 그룹에만 있으면 Sync 블록은 **no-op**(아무 일도 하지 않음)으로 동작합니다.
</Callout>

## 진행 순서

1. [플래너 생성 / 워크스페이스 추가](/docs/planner/create-and-workspaces) — 새 플래너를 만들고, 사용할 워크스페이스를 고릅니다. 워크스페이스를 추가하면 그룹과 가로 타임라인이 자동으로 만들어집니다.
2. [블록 추가 / 플랜 실행](/docs/planner/blocks-and-run) — 그룹별로 실행할 블록을 추가·정렬한 뒤 **전체 실행**(모든 그룹 병렬) 또는 **이 그룹만 실행**으로 시작하고 진행 상황을 모니터링합니다.
3. [플래너 내보내기](/docs/planner/export) — 완성된 플래너를 체크포인트 + 런타임 코드가 포함된 독립 실행 zip으로 묶어 EasyTrainer 없이 도커로 돌릴 수 있습니다 (선택 사항).

## 다음 단계

플래너로 여러 체크포인트를 엮어 동작이 끊이지 않게 만들었다면, 부족한 단계를 발견할 때마다 [데이터 수집](/docs/data-collection)으로 돌아가 더 많은 시연을 모으거나 [데이터 증강](/docs/datasets/augment)으로 학습 샘플을 늘릴 수 있습니다.
