ROBOTIS OMY는 **DYNAMIXEL-Y 시리즈 액추에이터** 기반의 **6-DOF 팔**로, ROS 2 `ros2_control` 위에서 동작하는 ROS 2 네이티브 로봇입니다. Easy Trainer는 OMY 모듈에서 다음 두 가지 variant를 지원합니다.

- **OMY-3M** — 6-DOF 팔만 있는 구성 (그리퍼 없음). 조인트: `joint1` ~ `joint6`.
- **OMY-F3M** — 6-DOF 팔 + **RH-P12-RN 2지(2-finger) 그리퍼** + **Intel RealSense D405 뎁스 카메라** 내장 구성. 조인트: `joint1` ~ `joint6` + `rh_r1_joint`.

<Callout type="info">
업스트림 OpenManipulator 제품군에는 OMY-L100(리더), OMY-AI3M(텔레오퍼레이션 세트) 같은 다른 variant도 있지만, **이 데이터 수집용 팔로워 모듈은 OMY-3M과 OMY-F3M만 다룹니다.** 업스트림 저장소는 [ROBOTIS-GIT/open_manipulator](https://github.com/ROBOTIS-GIT/open_manipulator)를 참고하세요.
</Callout>

## 사전 조건

- [모듈 추가](/docs/modules)에서 **OMY** 모듈 설치
- OMY 로봇 PC의 온보드 Dynamixel 버스가 동작 중인 상태
- 모터 버스에 전원이 인가된 상태
- OMY 로봇 PC에서 온보드 UART 포트가 보이는지 확인:

  ```bash
  ls /dev/ttyAMA*
  ```

  OMY-3M은 팔 버스 하나(`/dev/ttyAMA2`), OMY-F3M은 팔 버스(`/dev/ttyAMA2`)와 그리퍼(엔드 유닛) 버스(`/dev/ttyAMA4`)를 사용합니다. 포트가 보이지 않으면 케이블 연결과 권한을 점검합니다.

## 하드웨어 연결

- OMY는 ROS 2 네이티브 로봇으로, `ros2_control`이 온보드 Dynamixel 버스를 직접 제어합니다.
- **OMY-3M**: 팔 버스(기본 `/dev/ttyAMA2`) 하나만 연결됩니다.
- **OMY-F3M**: 팔 버스(기본 `/dev/ttyAMA2`)와 별도로 그리퍼(엔드 유닛) 버스(기본 `/dev/ttyAMA4`)가 연결됩니다. RH-P12-RN 2지 그리퍼와 RealSense D405 카메라가 팔 끝에 내장되어 있습니다.
- 포트가 출고 기본값과 다르면 등록 시 **Serial Port**(팔) / **Gripper Port**(F3M 그리퍼) 값을 바꿔 지정할 수 있습니다.

## 원격 SSH 운용

OMY는 보통 로봇 자체 PC(온보드)에서 ROS 드라이버를 실행하고, Easy Trainer가 설치된 PC와는 이더넷으로 연결되어 **같은 ROS 도메인에서 토픽만 주고받는** 방식으로 운용합니다. 이 경우 로봇 등록 시 SSH 접속 정보를 입력하면 Easy Trainer가 로봇 PC의 드라이버를 원격으로 기동·제어합니다.

### 사전 준비

- **인증 방식**: 로봇 등록 화면에 **SSH 비밀번호** 필드가 있습니다. 두 가지 방식 중 하나로 인증합니다.
  - **비밀번호 인증**: **SSH 비밀번호** 필드에 로봇 PC 계정의 비밀번호를 입력합니다.
  - **키 기반 인증**: **SSH 비밀번호** 필드를 비워 두면 SSH 키 인증을 사용합니다. 이 경우 Easy Trainer 호스트의 SSH 공개키를 로봇 PC에 미리 등록해 두어야 합니다.

    ```bash
    ssh-copy-id root@omy-SNxxxx.local
    ```

- **네트워크**: Easy Trainer 호스트와 로봇이 **같은 `ROS_DOMAIN_ID`(OMY는 30)** 여야 토픽이 보입니다. 두 장비를 같은 네트워크(서브넷)에 두는 것을 권장합니다.

<Callout type="warning">
입력한 SSH 비밀번호는 로봇 설정에 저장됩니다. 보안을 위해 가능하면 **키 기반 인증** 방식을 권장합니다.
</Callout>

### 등록 시 입력값

로봇 등록(로봇 페이지)에서 다음 필드를 입력하면 **원격 모드**로 동작합니다. 세 필드를 모두 비워두면 **로컬(직접 연결) 모드**로 동작합니다.

- **SSH Host**: 로봇 PC의 주소 (예: `omy-SNxxxx.local` 또는 IP)
- **SSH User**: 로봇 PC 접속 사용자 (기본값 `root`)
- **SSH Port**: 로봇 PC SSH 포트 (기본값 `22`)
- **SSH 비밀번호**: 로봇 PC 계정의 비밀번호. 키 기반 인증을 사용하려면 비워 둡니다.

### 동작 방식

로봇을 처음 실행하면 Easy Trainer가 로봇 PC에 SSH로 접속해 필요한 준비 작업(로봇 PC의 docker 컨테이너 기동 등)을 자동으로 수행한 뒤 드라이버를 실행합니다. 이미 준비된 단계는 건너뛰므로, 최초 1회만 시간이 걸리고 이후에는 빠르게 기동됩니다.

<Callout type="warning">
첫 기동은 로봇 PC의 컨테이너 준비로 **수 분**이 걸릴 수 있습니다. 카드가 ONLINE으로 바뀔 때까지 기다려 주세요.
</Callout>

## 등록 입력값

[공통 절차](/docs/robots#공통-절차) Step 2에서 다음 값을 입력합니다. 원격으로 운용하는 경우 위의 [원격 SSH 운용](#원격-ssh-운용) 필드를 함께 입력합니다.

### OMY-3M (팔만)

- **Robot Type**: `omy_3m`
- **Serial Port**: `/dev/ttyAMA2` (기본값. 팔 버스 포트가 다르면 변경)

저장하면 내부적으로 다음과 같은 매니페스트가 적용됩니다.

- **조인트 (총 6개)**: `joint1` ~ `joint6`
- **컨트롤러**: `JointGroupPositionController` 하나로 팔을 구동 (`/joint_group_position_controller/commands` 토픽으로 publish)
- **상태 토픽**: `sensor_msgs/JointState` (`/joint_states`)
- **IK**: 사용 가능 (엔드 이펙터 위치 제어)

### OMY-F3M (팔 + 그리퍼 + 카메라)

- **Robot Type**: `omy_f3m`
- **Serial Port**: `/dev/ttyAMA2` (기본값. 팔 버스 포트)
- **Gripper Port**: `/dev/ttyAMA4` (기본값. 그리퍼(엔드 유닛) 버스 포트)

저장하면 내부적으로 다음과 같은 매니페스트가 적용됩니다.

- **조인트 (총 7개)**: `joint1` ~ `joint6` + `rh_r1_joint`(그리퍼)
- **컨트롤러**: `JointGroupPositionController` 하나로 팔 + 그리퍼를 함께 구동 (`/joint_group_position_controller/commands` 토픽으로 publish)
- **상태 토픽**: `sensor_msgs/JointState` (`/joint_states`)
- **IK**: 사용 가능 (그리퍼 조인트는 IK 계산에서 잠금 처리)

토글을 ON 한 뒤 RobotPendant 로 각 조인트를 한 칸씩 움직여 회전 방향이 올바른지 확인합니다.

## 트러블슈팅

- **카드는 ONLINE인데 움직이지 않습니다.** `/joint_group_position_controller/commands` 토픽이 publish 되는지 `ros2 topic echo`로 확인하세요. ros2_control 컨트롤러가 spawn 되지 않은 상태일 수 있습니다.
- **OMY-F3M 그리퍼만 반응이 없습니다.** **Gripper Port**(기본 `/dev/ttyAMA4`)가 올바른지 확인하세요. 팔 버스와 그리퍼(엔드 유닛) 버스는 서로 다른 온보드 UART를 사용합니다.
- **모터가 검출되지 않습니다.** 모터 전원과 `/dev/ttyAMA2`(팔) / `/dev/ttyAMA4`(F3M 그리퍼) 포트를 차례로 점검합니다. 포트가 출고 기본값과 다르면 등록 화면에서 **Serial Port** / **Gripper Port** 값을 실제 포트로 맞추세요.
