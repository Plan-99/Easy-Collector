ROBOTIS AI Worker는 **양팔 7-DOF + 그리퍼** 구성의 휴머노이드형 플랫폼으로, 로봇 내부의 **NVIDIA Orin PC**에서 ROS 드라이버가 동작합니다. Easy Trainer는 네트워크로 로봇 PC에 **원격 SSH 접속**해 양팔을 하나의 트래젝토리 명령 스트림으로 구동하고, 두 장비는 **같은 ROS 도메인에서 토픽만 주고받습니다**. Easy Trainer는 AI Worker 모듈에서 다음 두 모델을 지원합니다.

- **FFW-BG2** — Robot Type `ffw_bg2`
- **FFW-SG2** — Robot Type `ffw_sg2`

두 모델 모두 **총 16개 조인트**(왼팔 7 + 왼 그리퍼 1, 오른팔 7 + 오른 그리퍼 1)를 제어하며, 역할은 `dual_arm`입니다.

<Callout type="info">
AI Worker는 **양팔(dual-arm)을 조인트 공간(joint space)에서 구동**하는 로봇입니다. **IK는 제공하지 않으므로**(엔드 이펙터 위치 제어 불가) 텔레오퍼레이션 기반 모방학습 데이터 수집에 주로 사용합니다. 머리·리프트·이동 베이스는 이 버전에서 다루지 않습니다(팔 전용).
</Callout>

## 사전 조건

- [모듈 추가](/docs/modules)에서 **AI Worker** 모듈 설치
- AI Worker 본체(NVIDIA Orin PC)가 켜져 있고 네트워크에 연결된 상태
- Easy Trainer 호스트와 로봇 PC가 **같은 `ROS_DOMAIN_ID`** 를 사용하는 상태

<Callout type="warning">
SSH 접속 대상은 반드시 **실제 AI Worker의 Orin PC**(사용자 `robotis`)여야 합니다. 드라이버 컨테이너는 arm64/CUDA 이미지이므로 일반 x86 PC에서는 동작하지 않습니다. 실제 출고 장비에는 `nvidia-container-toolkit`이 미리 설정되어 있습니다. **Orin에서 `apt upgrade`는 실행하지 마세요(ROBOTIS 경고).**
</Callout>

## 원격 SSH 운용 (필수)

AI Worker는 **로컬(직결) 모드가 없습니다.** 반드시 SSH 필드를 입력해야 하며, Easy Trainer가 로봇 PC의 Orin에 원격 접속해 드라이버를 자동으로 기동·제어합니다. 사용자가 컨트롤러를 직접 띄우거나 토픽을 수동으로 연결할 필요는 없습니다.

### SSH 인증

로봇 등록 화면에 **SSH 비밀번호** 필드가 있습니다. 두 가지 방식 중 하나로 인증합니다.

- **비밀번호 인증**: **SSH 비밀번호** 필드에 로봇 PC 계정의 비밀번호를 입력합니다. ROBOTIS 출고 기본값은 사용자 `robotis`, 비밀번호 `root` 입니다.

  ```bash
  ssh robotis@ffw-SNxxxx.local   # 비밀번호 root
  ```

- **키 기반 인증**: **SSH 비밀번호** 필드를 비워 두면 SSH 키 인증을 사용합니다. 이 경우 Easy Trainer 호스트의 SSH 공개키를 로봇 PC에 미리 등록해 두어야 합니다.

<Callout type="warning">
입력한 SSH 비밀번호는 로봇 설정에 저장됩니다. 보안을 위해 가능하면 **키 기반 인증** 방식을 권장합니다.
</Callout>

### 등록 시 입력값

[공통 절차](/docs/robots#공통-절차) Step 2에서 다음 값을 입력합니다.

- **Robot Type**: `ffw_bg2` 또는 `ffw_sg2`
- **SSH Host**: 로봇 PC의 주소 (예: `ffw-SNxxxx.local` 또는 IP)
- **SSH User**: 로봇 PC 접속 사용자 (기본값 `robotis`)
- **SSH Port**: 로봇 PC SSH 포트 (기본값 `22`)
- **SSH 비밀번호**: 로봇 PC 계정의 비밀번호 (출고 기본값 `root`). 키 기반 인증을 사용하려면 비워 둡니다.

### 동작 방식

로봇을 처음 실행하면 Easy Trainer가 로봇 PC의 Orin에 SSH로 접속해 준비 작업을 자동으로 수행합니다.

1. `~/ai_worker`에 저장소가 없으면 [ROBOTIS-GIT/ai_worker](https://github.com/ROBOTIS-GIT/ai_worker)(branch `jazzy`)를 git으로 내려받습니다(최초 1회).
2. 로봇 PC의 docker 컨테이너 `ai_worker`(ROS 2 Jazzy)를 기동합니다. 첫 기동에서는 Orin(arm64)용 큰 이미지를 준비하므로 **수 분**이 걸립니다.
3. 컨테이너 안에서 드라이버를 실행합니다. Easy Trainer는 **양팔 16개 조인트를 하나의 트래젝토리 명령 스트림**으로 구동하고, 상태는 `/joint_states`로 들어옵니다.

이미 완료된 단계는 건너뛰므로, 최초 1회만 시간이 걸리고 이후에는 빠르게 기동됩니다.

<Callout type="warning">
첫 기동은 로봇 PC의 컨테이너 준비로 **수 분**이 걸릴 수 있습니다. 카드가 ONLINE으로 바뀔 때까지 기다려 주세요.
</Callout>

저장 후 카드의 토글을 ON 한 뒤 RobotPendant로 각 조인트를 한 칸씩 움직여 양팔의 회전 방향이 올바른지 확인합니다.

## 트러블슈팅

- **카드가 ONLINE으로 바뀌지 않습니다.** 먼저 Easy Trainer 호스트에서 로봇 PC(Orin)로 `ssh robotis@ffw-SNxxxx.local`이 되는지 확인합니다. 비밀번호 또는 키, 사용자(`robotis`), 포트를 점검합니다.
- **카드는 ONLINE인데 토픽이 보이지 않습니다.** Easy Trainer 호스트와 로봇 PC의 **`ROS_DOMAIN_ID`가 동일**한지 확인합니다. 두 장비를 같은 네트워크(서브넷)에 두는 것을 권장합니다.
- **첫 기동이 오래 걸립니다.** 로봇 PC에서 `ai_worker` 컨테이너가 떠 있는지 확인합니다. 최초 1회는 저장소 클론과 arm64 이미지 준비로 수 분이 걸립니다.
- **컨테이너가 GPU를 인식하지 못합니다.** SSH 대상이 실제 AI Worker Orin인지 확인합니다. `nvidia-container-toolkit`이 필요하며, 실제 출고 장비에는 기본 설정되어 있습니다. 일반 x86 PC에서는 드라이버 이미지가 동작하지 않습니다.
